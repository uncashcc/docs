---

## Response

<ResponseField name="success" type="boolean">
  Indicates if the request was successful
</ResponseField>

<ResponseField name="message" type="string">
  Success message
</ResponseField>

<ResponseField name="data" type="array">
  Array of available BIN objects
  <Expandable title="BIN object">
    <ResponseField name="id" type="string">BIN identifier (UUID)</ResponseField>
    <ResponseField name="bin" type="string">Bank Identification Number (6 digits)</ResponseField>
    <ResponseField name="network" type="string">Card network (e.g., "master", "visa")</ResponseField>
    <ResponseField name="threeDS" type="boolean">3D Secure support enabled</ResponseField>
    <ResponseField name="wallet" type="boolean">Digital wallet support enabled</ResponseField>
    <ResponseField name="currency" type="string">Supported currency (e.g., "USD")</ResponseField>
    <ResponseField name="country" type="string">Country code (e.g., "HK", "US")</ResponseField>
    <ResponseField name="initial_balance" type="number">Initial balance added to card upon creation</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="timestamp" type="string">
  ISO 8601 timestamp
</ResponseField>

## BIN Properties Explained

- **network**: The card network provider (Mastercard, Visa, etc.)
- **threeDS**: Whether the BIN supports 3D Secure authentication for enhanced security
- **wallet**: Whether the BIN supports digital wallet features (Apple Pay, Google Pay, etc.)
- **initial_balance**: The starting balance that will be added to the card when created with this BIN
- **country**: The issuing country for the BIN

<Info>
  **BIN Selection**: Choose the appropriate BIN ID when creating cards. Consider the `network`, `threeDS`, `wallet`, and `country` properties based on your use case. The `initial_balance` will be automatically deposited to the card upon creation.
</Info>

<Warning>
  Only BINs with `status: "enabled"` are returned by this endpoint. Disabled BINs cannot be used for card creation.
</Warning>
