---
title: "Freeze Card"
api: "PATCH https://api.uncash.io/api/v1/cards/{id}/freeze"
description: "Temporarily freeze a card to block all transactions"
---

## Path Parameters

<ParamField path="id" type="string" required>
  The card ID (UUID) to freeze
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Indicates if the card was frozen successfully
</ResponseField>

<ResponseField name="message" type="string">
  Success message
</ResponseField>

<ResponseField name="data" type="object">
  <Expandable title="properties">
    <ResponseField name="card_id" type="string">Card ID (UUID) that was frozen</ResponseField>
    <ResponseField name="status" type="string">New card status ("frozen")</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="timestamp" type="string">
  ISO 8601 timestamp
</ResponseField>

## Behavior

- **Transaction Blocking**: All card transactions will be declined while frozen
- **Balance Preserved**: Card balance remains unchanged
- **Reversible**: Card can be unfrozen using the unfreeze endpoint
- **Provider Update**: Freeze status is applied at the provider level

<Info>
  **Use Case**: Freeze your card temporarily if you suspect unauthorized access or lose your physical card details.
</Info>

## Validation Rules

- Card must be in "active" status
- Card must have a valid `provider_id` (not 'temp')
